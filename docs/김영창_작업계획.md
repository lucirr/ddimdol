## 구현 단계 계획

### Phase 1: 중앙 바이너리 릴리즈 파이프라인 구축 (2026-06 ~ 07)

**목적**: 폐쇄망 에지 노드에 안전하게 소프트웨어를 배포하기 위한 중앙 릴리즈 파이프라인 구현

**구현 항목**:
- 보안점검 파이프라인 구성 (SBOM 생성 + CVE 스캔 + 서명)
  - `binup` 기반 단방향 터널링 구조로 패키지 반입
  - 반입 파일에 대한 무결성 검증(해시) 및 서명 확인 모듈
- 보안 점검 통과 시 에지 노드 알림(Notification) 발송 메커니즘
- 릴리즈 버전 메타데이터 관리 (Catalog 연동)

**주요 기술**:
- Gitea/Harbor 기반 내부 레지스트리
- ArgoCD 기반 GitOps 릴리즈 관리
- Helm Chart + Container Image 패키징

**성공 기준**:
- 보안점검 파이프라인 통과 후 에지 노드에 알림 전달 확인
- 릴리즈 패키지 무결성 검증 100% 통과

---

### Phase 2: 관리자 승인 기반 업데이트 시스템 구현 (2026-08 ~ 10)

**목적**: Approval Gate — 중앙에서 배포하더라도 현장 관리자의 최종 승인 후에만 설치되는 인적 통제 프로세스 구현

**구현 항목 (2026-08~09)**:
- 업데이트 알람 UI: 중앙 콘솔 → 에지 관리자에게 신규 패키지 알림
- 관리자 승인(Approve) 워크플로우 백엔드 API
  - 승인/거부/연기 상태 관리
  - 승인 이력 감사 로그(Audit Log) 저장
- Agent Pull 방식 다운로드: 관리자 승인 확인 후 에지 노드가 패키지 Pull

**구현 항목 (2026-09~10)**:
- 카탈로그 자동 업데이트 활성화 설정 UI
  - 관리자 승인 시 카탈로그 내 모델/앱 버전 자동 갱신
  - 롤백(Rollback) 기능: 이전 버전으로 즉시 복구
- 업데이트 스케줄링 (예약 업데이트, 긴급 패치 Push 허용)

**주요 기술**:
- Web 기반 관리 콘솔 (React/Vue + REST API)
- Kubernetes Operator 패턴으로 에지 업데이트 상태 관리
- RKE2 기반 에지 클러스터 연동

**성공 기준**:
- 중앙 배포 → 승인 → Agent Pull → 설치/롤백 사이클 50회 반복 시 성공률 ≥ 99%

---

### Phase 3: 에지-중앙 관제 연동 구현 (2026-11 ~ 12)

**목적**: 에지 노드의 실시간 상태를 중앙에서 모니터링하고 에지에서도 관제 기능 활성화

**구현 항목**:

*[에지 운영 관리 측]*
- 에지 노드 상태 수집 에이전트 개발
  - 노드 up/down 이벤트 감지 및 중앙 전송
  - CPU/Memory/Disk/GPU 등 상세 자원 상태 공유
  - 수집 주기: 10초 (Kubernetes Device Plugin + Health Check)

*[중앙 에지 관리 측]*
- 중앙 관제 콘솔 에지 상태 뷰 개발
  - 에지 노드 목록, 상태 대시보드
  - 알람 임계치 설정 및 알림 발송
- 에지→중앙 단방향 메트릭 전송 (outbound only, Air-Gapped 준수)

**주요 기술**:
- Prometheus + Grafana 에이전트 (push 방식)
- Kubernetes Device Plugin API
- 에지 outbound 터널 (송원빈 구축 인프라 연동)

**성공 기준**:
- 노드 장애 감지 후 중앙 알람까지 지연 ≤ 10초
- SW 장애 복구 SLA ≤ 5분

---

### Phase 4: JIT 원격 접속 및 공동 대응 체계 구현 (2027-01 ~ 02, 공동: 송원빈)

**목적**: 현장 전문가 없이도 원격으로 에지 장애를 해결하는 JIT(Just-In-Time) 원격 대응 시스템

**구현 항목** (김영창 + 송원빈 협업):
- 중앙 → 에지 원격 접속 코드 발급 시스템
  - 시간 제한 원격 세션 토큰 발급
  - 사용자 PC 화면 공유 형태의 원격 지원
- 에지 → 중앙 접속 허용 화이트리스트 관리
- 원격 접속 세션 감사 로그 (접속자, 시간, 수행 명령 기록)
- 공동 대응 체계: 중앙 전문가 + 현장 관리자 동시 접속 지원

**역할 분담**:
- 김영창: 승인/인가 시스템, 세션 토큰 관리, 감사 로그
- 송원빈: 터널링 인프라, 네트워크 연결 설정

**성공 기준**:
- 원격 접속 요청부터 세션 활성화까지 ≤ 30초
- 세션 기록 100% 감사 로그 보존

---

### Phase 5: 에지 배포 성공률 성능 시험 (2027-05 ~ 06)

**목적**: 온프레미스 배포 성공률 KPI(99%) 달성 검증

**시험 내용**:
- 시험 환경: 중앙 콘솔 → 에지 노드, USB 기반 물리 매체 패키지 반입 후 배포
- 시험 조건: 연속 50회 배포 (패키지 크기 500MB~2GB, Helm Chart + Container Image)
- 측정 항목:
  - 배포 성공 횟수 / 전체 시도 횟수
  - 롤백 정상 동작 확인
  - 승인 게이트 통과 정확도
- 측정 도구: 자체 배포 자동화 스크립트 + 로그 분석

**최종 검증**:
- 공인인증기관(TTA) 성능 시험 지원
- KPI: 배포 성공률 ≥ 99%
